# Contributing to StructuraMiner

Thank you for your interest in contributing to StructuraMiner! We welcome bug reports, feature requests, and pull requests.

## Getting Started

1. **Fork the repository** on GitHub
2. **Clone your fork locally**:
   ```bash
   git clone https://github.com/lovekaushik899/StructuraMiner.git
   cd StructuraMiner
   ```

3. **Create a virtual environment**:
   ```bash
   python -m venv .venv
   source .venv/bin/activate  # On Windows: .venv\Scripts\activate
   ```

4. **Install development dependencies**:
   ```bash
   pip install -e .[dev]
   ```

## Development Workflow

1. **Create a feature branch**:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** and test thoroughly

3. **Commit your changes** with clear messages

4. **Push to your fork** and submit a Pull Request

## Code Style

- Follow PEP 8 conventions
- Use descriptive variable names
- Add docstrings to new functions
- Keep lines under 100 characters where practical

## Testing

Test your changes with various PDB files before submitting PRs.
